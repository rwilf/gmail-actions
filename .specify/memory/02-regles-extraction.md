# Règles métier — Extraction des actions

## Définition d’une action
Une action est une tâche concrète, exécutable par un humain,
pouvant être formulée sous la forme :
> Verbe d’action + complément

Exemples :
- Répondre au mail
- Donner son avis sur la formation
- Valider le document
- Proposer des créneaux
- Faire un retour

## Cas explicites
- Impératifs : "Merci de me confirmer", "Peux-tu envoyer"
- Questions orientées action : "Peux-tu me dire si", "As-tu eu le temps de"

## Cas implicites (critiques)
Même si l’action n’est pas formulée directement :
- Demande de retours
- Demande d’avis
- Demande de validation
- Demande de relecture
→ doivent être transformées en action concrète

Exemple :
"Quels sont vos retours ?"
→ Action : "Donner ses retours"

## Langue et forme
- Indépendant de la langue (français / anglais minimum)
- Indépendant de la personne grammaticale (tu, nous, vous, ...)
- Forme neutre et générique (infinitif)

## Déduplication
- Une action identique présente plusieurs fois dans un thread
  ne doit apparaître qu’une seule fois.
- Les citations, forwards et réponses ne doivent pas créer
  de nouvelles actions.

## Portée
- Les actions concernent uniquement le destinataire du mail
- Les actions destinées à d’autres personnes doivent être ignorées

## Qualité attendue
- Peu d’actions mais pertinentes
- Mieux vaut manquer une action que créer du bruit
